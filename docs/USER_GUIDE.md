# QQ聊天增强系统 用户手册

> 本手册面向最终用户，介绍 QQ 智能助手管理平台的功能使用。前端基于 Next.js 16 (App Router) + React 19 + shadcn/ui 构建。
> 部署后通过浏览器访问前端（默认端口 5000，本地开发 3000），前端 `/api/*` 自动代理到后端 FastAPI（端口 8000）。

---

## 1. 系统简介

QQ 聊天增强系统是一个集 **对话管理、知识库 RAG 检索、LoRA 角色微调、模型推理** 于一体的管理平台。核心能力：

- **多角色对话**：通过 LoRA 适配器切换不同角色人设（如胡桃、minamo），基于 Qwen2.5-7B-Instruct-AWQ 量化模型推理。
- **知识库 RAG**：上传文档/ZIP，自动分块向量化，对话时按意图路由检索并注入背景设定。
- **LoRA 训练**：内置对话数据生成器，可基于角色描述批量生成训练数据并一键发起 LoRA 微调。
- **统一管理面板**：仪表盘、历史记录、知识库、训练、LoRA、设置等模块集中管理。

### 1.1 服务架构概览

```
浏览器 ──> Next.js 前端(5000) ──/api/*──> FastAPI 后端(8000) ──> vLLM 推理(8001)
                                            ├── SQLite (qq_assistant.db)
                                            ├── Faiss 向量库
                                            └── Redis [可选]
```

## 2. 登录与注册

### 2.1 首次登录

1. 浏览器打开前端地址（如 `http://localhost:5000`）。
2. 未登录时页面会提示「请先登录」，点击「前往登录」跳转 `/login`。
3. 首次使用点击注册，填写：
   - **用户名**：2–50 字符
   - **密码**：8–100 字符
4. 注册成功后系统自动签发 JWT 并写入 httpOnly Cookie（有效期 24 小时），无需手动管理 Token。

### 2.2 登录机制说明

- 登录态通过 httpOnly Cookie 自动携带，前端请求无需手动附加 Token。
- Token 过期或被注销（登出）后需重新登录。
- 生产环境（HTTPS）Cookie 会启用 `Secure` 标志。
- 后端同时支持 `X-API-Key` 头认证（用于脚本/服务间调用，由管理员配置 `API_KEYS` 环境变量），普通用户使用账号登录即可。

## 3. 仪表盘（首页）

> 路由：`/`（首页即仪表盘，由 `src/app/page.tsx` 渲染 `DashboardClient` 组件）。登录后默认进入。

仪表盘提供系统运行状态的全局视图：

### 3.1 统计卡片

| 卡片 | 含义 |
| --- | --- |
| 今日回复数 | 当天 0 点至今的回复消息总数 |
| 平均响应时间 | 今日所有回复的平均耗时（秒） |
| 活跃会话 | 今日有消息交互的唯一会话数 |
| 模型负载 | 基于今日回复数估算的负载百分比 |

### 3.2 今日活动趋势

24 小时活动折线图（每 2 小时一个数据点），展示消息数与回复数走势。该图表按需懒加载。

### 3.3 系统状态面板

- **模型信息**：基座模型（Qwen2.5-7B）、当前激活的 LoRA（无则显示「基础模型（无LoRA）」）。
- **系统资源**：CPU 使用率、内存使用量。
- **服务状态**：Backend API、NoneBot Bot（端口 8081）、Model Service、In-Memory DB、NapCat（端口 6099）的运行状态指示灯（运行中/连接中/已停止）。

### 3.4 快捷操作

| 入口 | 功能 |
| --- | --- |
| 测试回复 | 打开聊天式测试对话框，可直接与当前 LoRA 角色对话（见 §4 对话） |
| 切换模型 | 弹窗列出所有 LoRA，一键激活某个 LoRA（见 §8 角色切换） |
| 重启服务 | 服务重启入口 |
| 管理会话 | 弹窗管理各会话的机器人开关（启用/停用某会话的自动回复） |

> 右上角刷新按钮可重新拉取统计与服务状态。

## 4. 对话功能

系统中的「对话」有两处入口：

### 4.1 仪表盘测试对话（TestChatDialog）

位于仪表盘「快捷操作 → 测试回复」。打开后是聊天式界面：

- 可选择使用哪个 LoRA 角色回复（默认用当前激活的 LoRA）。
- 输入消息发送，后端调用 `POST /api/generate` 推理，返回回复文本、模型名、耗时。
- 该对话用于测试角色效果，消息会写入历史记录。

### 4.2 QQ 机器人自动回复

部署 NoneBot2 机器人后，QQ 群聊/私聊消息会触发自动回复：

- 由「设置 → 机器人」控制自动回复、群聊回复、私聊回复的总开关。
- 「管理会话」可针对单个会话单独关闭机器人。
- 回复时若开启知识库，会先做意图检测与 RAG 检索，再注入背景设定生成回复。

### 4.3 生成参数

对话生成时使用「设置 → 模型」中的参数（温度、最大生成长度等），RAG 命中时温度会自动降到 ≤0.5 以更忠实于检索内容。

## 5. 历史记录

> 路由：`/history`。需登录。查看和管理所有对话历史。

### 5.1 筛选与搜索

顶部筛选区支持：

- **关键词搜索**：匹配消息内容与回复内容。
- **会话类型**：全部 / 群聊 / 私聊。
- **LoRA 模型**：从历史消息中提取实际出现的 LoRA 名称进行过滤。
- **会话名称**：按群聊名/私聊名称过滤。

### 5.2 列表与操作

表格列：时间、用户、会话（类型+名称）、消息预览、LoRA 模型、耗时、操作。

| 操作 | 说明 |
| --- | --- |
| 查看（眼睛图标） | 弹窗查看完整对话详情（用户消息 + 机器人回复、模型、LoRA、耗时） |
| 删除（垃圾桶图标） | 二次确认后删除单条记录 |
| 导出数据 | 将当前筛选结果导出为 CSV（含 BOM，Excel 兼容） |
| 删除全部 | 按当前筛选条件批量删除（二次确认，不可撤销） |

> 顶部显示「共 N 条记录（当前筛选 M 条）」，N 为数据库全表计数。

## 6. 知识库

> 路由：`/knowledge`。需登录。管理知识库、文件夹、文档，并提供 RAG 检索。

### 6.1 界面布局

- **左侧**：知识库列表 + 文件夹列表（选中某知识库后显示其文件夹）。可创建/删除知识库与文件夹。
- **右侧**：文档内容区，含搜索框、统计卡片、文档列表。

### 6.2 知识库与文件夹

- **知识库**：文档的顶层容器（如「原神知识」），可创建/删除（删除级联清除文件夹与文档）。
- **文件夹**：知识库下的分类容器，删除文件夹时文档移至「未分类」。

### 6.3 添加文档的三种方式

| 方式 | 说明 |
| --- | --- |
| 手动添加 | 点击「添加文档」，填写标题与内容 |
| 上传 ZIP | 点击「上传ZIP」，上传压缩包；ZIP 内「顶层目录名/文件名.txt」结构会自动转为文件夹+文档，支持 `.txt/.md/.json`，单文件上限 100MB |
| 扫描文件夹 | 点击「扫描文件夹」，扫描服务器 `backend/knowledge_bases/` 目录，预览目录树后一键导入（支持 `.txt/.md/.json/.csv/.html/.xml`） |

> 文档创建/更新时会自动分块（`simple_text_split`），并把 `[知识库名/文件夹名] 标题: 内容` 作为检索文本写入 Faiss 向量库。

### 6.4 知识库检索（RAG）

- 顶部搜索框输入查询，点击搜索后展示相关片段（含相关度百分比）。
- 后端检索链路：RAGHelper 两阶段（向量+BM25 混合 → Cross-Encoder 精排）→ 向量检索 → 关键词匹配（逐级回退）。
- 支持按知识库名称过滤检索结果（意图分类器训练后可自动路由到对应知识库）。

### 6.5 统计卡片

未搜索时展示：文档总数、向量条目数（约等于分块数）、总字符数。

### 6.6 意图分类器训练

知识库模块还提供意图分类器训练能力（用于对话时自动判断该检索哪个知识库）：

- 生成训练样本（LLM 基于知识库文档生成）。
- 审查/编辑/删除/批量保存样本。
- 训练多分类模型，查询训练进度，取消训练。
- 设置参与检索的知识库。

> 该功能进阶使用，普通用户只需确保知识库文档已导入即可使用基础 RAG。

## 7. 训练（LoRA 微调）

> 路由：`/training`。需登录。包含数据集管理、对话数据生成、训练任务、已保存对话管理。

页面通过 Tab 切换不同子功能，表单数据会自动持久化（刷新不丢失）。

### 7.1 数据集管理

- **创建数据集**：填写数据集名称、选择人物风格、可选自定义 prompt，粘贴对话数据（或从文件导入）创建。
- **扫描文件夹**：扫描服务器目录发现数据集子文件夹，预览后导入到训练目录（路径须在 backend 或 `/root/autodl-tmp` 下）。
- **导出**：将数据集导出为 ZIP 下载（便于上传到服务器训练）。
- **预定义风格**：列出可用的人物风格模板。

### 7.2 快速生成对话数据

核心功能：基于角色描述自动生成 LoRA 训练对话。

1. 填写 **角色描述**（如「胡桃，原神往生堂第七十七代堂主…」）。
2. 可选填写 **角色风格** 与 **额外要求**。
3. 设置 **生成数量**（默认 500）。
4. 点击生成，系统会：
   - 网络搜索角色背景信息丰富上下文。
   - 按轮次分布策略（1–8 轮，加权分布）生成多轮对话。
   - 覆盖 12 种场景类型（日常问候、闲聊吐槽、情感倾诉、角色扮演等）。
   - 严格验证 user/assistant 交替配对，截断到目标轮数。
5. 生成过程中实时显示进度（批次、已生成数、百分比），支持取消。
6. 生成完成后可预览、编辑、筛选、勾选对话，并保存到「已保存对话」。

> 同一时刻仅允许一个生成任务运行；断线重连时会自动恢复进度。模型为 mock 模式时无法生成（需在设置中配置有效模型提供商）。

### 7.3 已保存对话管理

- 保存的对话以列表展示，含角色描述、风格、对话数、轮次/场景统计。
- 支持查看、删除单条对话、删除整个保存项。
- **从已保存对话创建数据集**：一键将保存的对话转换为标准训练数据集。

### 7.4 启动 LoRA 训练

1. 选择数据集与 GPU 训练配置（如 `qwen2.5-7b-3090`，区分 RTX 3090 24GB / RTX 4060 8GB）。
2. 填写新 LoRA 名称。
3. 可通过参数编辑器自定义训练配置（rank、batch、学习率等，覆盖基础配置）。
4. 启动训练前系统会检查 GPU 显存（≥8GB）或系统内存（≥16GB），不足时返回 507 并建议导出数据集到服务器训练。
5. 训练任务列表实时展示状态（pending/training/completed/failed/cancelled）与进度，可取消任务。

### 7.5 训练完成后

- 训练产出的 LoRA 适配器会落到 `backend/loras/` 目录。
- 前往 LoRA 管理页（见 §8）点击「扫描新LoRA」即可注册到数据库并激活使用。

## 8. 角色切换（LoRA 适配器）

> LoRA 管理路由：`/lora`。需登录。角色切换也可在仪表盘「切换模型」入口完成。

### 8.1 LoRA 列表

- 卡片式展示所有 LoRA，含名称、描述、风格类型、模型大小、训练进度（已训练步数/总步数）、创建时间。
- Tab 切换：全部 / 已启用 / 已停用。
- **同一时间只能启用一个 LoRA** 进行推理，激活新的会自动停用其他。

### 8.2 操作

| 操作 | 说明 |
| --- | --- |
| 扫描新 LoRA | 扫描 `backend/loras/` 目录，自动发现并注册新适配器（读取 `adapter_config.json`，支持 `final/` 子目录），更新已有记录的 size/步数 |
| 启用/停用开关 | 切换 LoRA 激活状态，激活后所有新消息使用该角色回复 |
| 删除 | 删除 LoRA 记录（二次确认，不可恢复） |

### 8.3 角色与系统提示词

每个 LoRA 角色在 `backend/bot/bot.py` 的 `LORA_REGISTRY` 中注册了对应的系统提示词，对话时会自动作为 system 消息注入。数据库中的 LoRA 名称到 vLLM 注册名有映射（如 `hutao_lora_7b`→`hutao`、`minamo_lora`→`minamo`），vLLM 侧通过启动参数 `--lora-modules` 预加载。

| 自动回复 | 总开关，关闭后机器人不自动回复 |
| 群聊回复 | 是否在群聊中自动回复 |
| 私聊回复 | 是否在私聊中自动回复 |
| 回复延迟 | 回复前等待秒数（0–10） |
| 默认回复模板 | 默认回复模板文本 |

### 9.3 模型（Model）

| 设置项 | 说明 |
| --- | --- |
| 模型提供商 | vLLM(本地GPU推理) / DeepSeek·OpenAI兼容API / Transformers+PEFT(本地) / Ollama(本地) / llama.cpp(本地) / Mock(测试) |

## 10. 常见问题 FAQ

**Q1：登录后页面空白或提示「请先登录」？**
A：检查后端服务是否正常运行（`/health` 探针）。Cookie 可能因域名/端口变化失效，重新登录即可。多实例部署时需统一 `JWT_SECRET`。

**Q2：对话没有角色感，回复像通用模型？**
A：① 确认已在 LoRA 管理页激活了对应角色 LoRA；② 检查 vLLM 是否通过 `--lora-modules` 预加载了该 LoRA；③ 在设置页「检查 vLLM 连接状态」确认实例健康。

**Q3：知识库检索没有命中？**
A：① 确认文档已创建并分块（知识库页统计的「向量条目」>0）；② 首次搜索会延迟重建 Faiss 索引，稍候即可；③ 确认设置页「启用知识库」开关已开启；④ 检查意图分类器是否误判不需要检索。

**Q4：训练启动失败提示显存不足？**
A：7B 模型训练至少需要 8GB GPU 显存或 16GB 系统内存。可在本地生成数据集后导出 ZIP，上传到 GPU 服务器训练；或降低 `lora_rank`、`per_device_train_batch_size` 等参数。

**Q5：对话生成卡住或进度不动？**
A：① 模型为 mock 模式时无法生成，请在设置中切换到有效提供商（如 vLLM 或 DeepSeek API）；② 同一时刻只允许一个生成任务，若有残留任务可使用「强制重置」；③ 断线重连会自动恢复进度。

**Q6：后端多 worker 导致训练状态丢失？**
A：训练/生成状态是模块级变量，多 worker 间不共享。后端必须以 `--workers 1` 启动。

**Q7：Redis 连接失败告警？**
A：Redis 为可选组件，未配置时后端自动降级为数据库直连模式，功能不受影响但性能略低。如需启用，安装 Redis 并在 `.env` 配置 `REDIS_URL=redis://localhost:6379/0`。

**Q8：修改了代码但行为没变？**
A：Python 缓存的 `.pyc` 文件可能导致修改不生效，清除 `backend/__pycache__` 并设置 `PYTHONDONTWRITEBYTECODE=1`。

**Q9：历史记录删除后还能恢复吗？**
A：不能。单条删除与批量删除均不可撤销，操作前请确认或先导出 CSV 备份。

**Q10：如何让某个 QQ 群不自动回复？**
A：在仪表盘「管理会话」中找到该会话，关闭其机器人开关即可单独停用。
